Proposed Solution

How It Works

  1. Connect to any SQL database or upload Excel/CSV
  2. Accept plain-English business questions
  3. Run three-pass validation pipeline (generate → review → validate)
  4. Execute query and return results in virtualized table
  5. Log all activity for compliance

Three-Pass Pipeline

Pass 1 - Generation: Claude Sonnet receives schema, column descriptions, and sample rows. Generates SQL with strict constraints (explicit JOINs, LIMIT 10, no SELECT *).

Pass 2 - Review: Second pass validates semantic correctness (JOIN conditions, GROUP BY completeness). Falls back to Pass 1 if review fails.

Pass 3 - Validation: SQLGlot AST validation ensures read-only (no INSERT/UPDATE/DELETE). Executes on target database.
